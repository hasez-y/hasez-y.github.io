# 序列化 & 反序列化

  • 序列化：将对象转化为字节序列的过程称之为序列化，
  • 反序列化：将字节序列转化为对象的过程称之为反序列化

# 序列化的作用

序列化的核心作用是对象状态的保存和重建，当需要保存到本地或网络传输对象时，就需要将对象序列化为字节然后网络传输或通过流保存到本地；况且对象是保存在 JVM 堆内存中的，只要该 JVM 关闭都会丢失对象。

# 为什么需要序列化

对内存的对象进行持久化或网络传输，这个时候都要将对象序列化和反序列化

# Java 序列化实现方式

  1. 实现 Serializable 接口
  2. 实现 Externalizable 接口

# 实现 Serializable 接口

将要序列化的类实现 java.io.Serizlizable 接口以启用其序列化功能，实现后该类以及子类都可序列化

# 序列化版本号 serialVersionUID

所有实现序列化的对象都必须要有个版本号，这个版本号可以由我们自己定义，当我们没有定义时，JDK 工具会按照对象的属性生成一个对应的版本号。

版本号有什么用？

版本号用来表明类的不同版本间的兼容性

序列化机制是通过运行时判断类的 serialVersionUID 来验证版本一致性的，在进行反序列化时，JVM 会把字节流中的 serialVersionUID 与本地类的版本号对比，如果一致则可以进行反序列化，否则会报版本不一致的异常。

JDK 自动生成和自定义的版本号有什么区别

如果不显示指定版本号，JVM 会在序列化根据属性生成版本号，如果修改对象时，那么就会导致生成的版本号新旧不一样而报错。而自定义版本号则不会报错。
