InnoDB 回表查询

首页

回表查询是由于在 InnoDB 查询记录时命中非聚集索引获得主键，在通过主键到聚集索引查询才能完成该次查询，这个过程称之为回表查询

在不懂聚集索引和非聚集索引之前，听起来是一脸懵逼的，下面就会讲什么是聚集索引和非聚集索引

# 聚集索引 和 非聚集索引

MySQL 存储引擎 InnDB 中规定，一个表必定有且仅有一个聚集索引，而非聚集索引由用户自定义个数。

InnDB 所用的索引数据结构是 B + 树，树的叶子节点存储行记录数据，这种索引称之为聚合索引，也叫主键索引；

而另一种索引是 树的叶子节点存储的是 主键值，这种索引称之为非聚合索引，也叫非主键索引、二级索引

# 回表查询

对于 InnoDB 来说，想要查询数据我们还需要根据主键再去聚集索引种查找，期间扫描了两遍索引树，这个过程称之为回表。

关于默认的聚集索引：

  1. 如果表定义了主键，那么该主键就是聚集索引
  2. 如果表没有定义主键，那么第一个 NOT NULL & UNIQUE 列是聚集索引
  3. 以上都不没有，那么 InnDB 会创建一个隐藏的 row-id 作为聚集索引

所以主键查询非常快，是因为有索引，直接定位行记录

# 非聚合索引一定要回表查询吗？

不一定，这涉及到查询语句所要求的字段是否全部命中索引，如果全命中索引，那么就不用回表查询。

一个索引包含所需查询的值，那么称之为 覆盖索引

假设一个查询只需要 name 值，而 name 是非聚合索引，那么该索引最后会获得 name 值和主键，这时就不需要回表查询了，因为 覆盖索引 了。

建立多个字段的联合索引时需要注意，在联合索引种，如果想要命中索引，需要按照建立索引时的字段顺序挨个使用，否则无法命中索引。
