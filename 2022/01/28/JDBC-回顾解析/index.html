# 什么是 JDBC

JDBC（Java Data Base Connectivity，java 数据库连接）是一种用于执行 SQL 语句的 Java API，可以为多种关系数据库提供统一访问，它由一组用 Java 语言编写的类和接口组成。

JDBC 提供了一种基准，据此可以构建更高级的工具和接口，JDBC 的实现是由数据库厂商提供，以驱动程序形式提供，使数据库开发人员能够编写数据库应用程序。例如 Mybatis

JDBC 是接口，而 JDBC 驱动才是接口的实现，没有驱动无法完成数据库连接！每个数据库厂商都有自己的驱动，用来连接自己公司的数据库。

# JDBC 核心接口介绍

JDBC 中的核心类有： DriveManager 、 Connection 、 Statement 、 ResultSet

# DriveManager

  1. 注册驱动 - 让 JDBC 知道要加载的驱动是哪个
  2. 获取 Connection - 与数据库的会话，代表一个数据库的物理连接

# Connection

Connection 对象表示连接，与数据库的通讯都是通过该对象展开的

  1. Connection 最为重要的一个方法就是获取 Statement 对象
  2. Statement 是用来向数据库发送 SQL 语句的，数据库会执行该语句
  3. 发送并执行 SQL 语句后，数据库会返回 ResultSet 结果集

# ResultSet

ResultSet 表示结果集，结果集是一个二维数组，有行有列，操作结果集要移动 ResultSet 内部的行光标，以及获取当前行上的每一列上的数据

# SQL 注入攻击

SQL 注入是比较常见的网络攻击方式之一，它不是利用操作系统的 BUG 来实现攻击，而是针对程序员编写时的疏忽，通过 SQL 语句，实现无账号登录，甚至篡改数据库。

用户故意传参时加入 SQL 语句来改变语句的意思从而执行一些特殊的操作，例如删库等，这种攻击叫做 SQL 注入攻击

JDBC 对此也有解决方案， PreparedStatement 利用预编译的机制将 sql 语句的主干和参数分别传输给数据库服务器，从而使数据库分辨出那些的 SQL 语句的主干，而那些是参数，这样即使参数中带有 SQL 的关键字，数据库也仅仅当数值使用，从而防止 SQL 注入的问题。

# Statment & PreparedStatement

  • PreparedStatement 可以使用预编译的 sql，而 Statment 只能使用静态的 sql

  • PreparedStatement 可以使用 sql 缓存区，效率比 Statment 高

  • PreparedStatement 可以有效防止 sql 注入，而 Statment 不能防止 sql 注入

最后有关一点是加载数据库驱动时是用反射来加载的，并不是通过 new 关键字，如果使用 new 会对具体的类产生依赖，后续更换数据库驱动，需要重新修改代码。而使用反射的方式，只需要在配置文件中，修改对应的驱动和 url 即可，达到解耦的效果
