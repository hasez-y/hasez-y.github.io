# 红黑树

R-B Tree，全称是 Red-Black Tree，又称为 “红黑树”，它一种特殊的二叉查找树。红黑树的每个节点上都有存储位表示节点的颜色，可以是红 (Red) 或黑 (Black)。

# 二叉查找树

二叉查找树特性 :

  • 左子树的所有节点的值都小于或等于它的根节点的值
  • 右子树的所有节点的值都大于或等于它的根节点的值
  • 左右子树也称为排序子树

二分查找树运用了二分查找的思想，查找所需的最大次数等同于二叉查找树的高度

但是还是存在缺陷，当二叉查找树插入新节点导致树两个边不平衡时，此时查找效率近乎线性查找，于此诞生红黑树。

# 红黑树

红黑树是一种自平衡的二叉查找树，具有二叉查找树基本特性，此外还有自己的特性。

  1. 节点是红色或黑色
  2. 根节点是黑色
  3. 每个叶子结点都是黑色的空结点（NIL 结点）
  4. 每个红色结点的两个子结点都是黑色。(从每个叶子到根的所有路径上不能有两个连续的红色结点)
  5. 从任一结点到其每个叶子的所有路径都包含相同数目的黑色结点

根据以上规则排列的红黑树，保证红黑树的自平衡，红黑树从根到叶子的最长路径不会超过最短路径的两倍

当插入或删除节点时，红黑树规则可能被打破，需要插入后调整红黑树

  • 变色
  • 左旋转
  • 右旋转

# 查找二叉树的删除

删除节点有三种情况，这里只介绍最复杂的情况，需要删除的节点有两个孩子，需要选则与需要删除节点值最接近的节点取代它，并删除取代它的节点

增删红黑树节点规则参考: https://zhuanlan.zhihu.com/p/143396578

# AVL 树和红黑树的区别

  • AVL 树是严格平衡的二叉树，要求左右子树高度不超过 1
  • 红黑树只要求任何路径长度不超过其他路径的长度 2 倍

根据以上特性得出，AVL 树查找效率更高，但是增删效率不如红黑树

更频繁查找使用 AVL 树，更多增删操作用红黑树

# HashMap 与 红黑树

JDK1.7 和 JDK1.8 对 HashMap 的实现做了很大的改动和优化，1.7 以及之前的 HashMap 是通过 数组 + 链表 实现，1.8 之后是数组 + 链表或红黑树实现的。
