# 什么是索引

生活中索引的使用无处不在，比如说：书本里的目录；读书时的座位号，考试编号都有类似索引的功能；生活的索引不言而喻，都是为了寻找而加上索引，更方便也更快捷；而数据库中的索引其实也是相同的道理，是为了提高查询效率。

索引是一种利用某种规则的数据结构与实际数据的关系加快数据查找的功能；索引数据节点中有着实际文件的位置，因为索引是根据特定的规则和算法构建的，在查找的时候遵循索引的规则可以快速查找到对应数据的节点，从而达到快速查找数据的效果；

总结来所有通过某规则数据结构和实际目标关联，根据特定规则算法快速寻址的功能都可以称之为索引

# InnoDB 的索引选择

MySQL 的默认存储引擎 InnoDB 使用 B+树 作为索引的存储结构，至于为什么使用 B+树 ，要对 B+树 的衍生过程做一些了解。

# 数组 or 链表

作为最基础的数据存储结构的数组和链表，是选用数组还是链表作为基本结构呢，这里需要分析它们的特性：

  • 数组：查找快、增删效率慢
  • 链表：查找慢、增删快

可以说这两种数据结构半斤八两，都并不完美，既然不完美，只能退而求其次了。由于数组的特性，也就是数组的操作会导致大量元素的迁移，且迁移的过程中是不能查找数据的，未迁移完的数据查询出来可能是错的，但是在数据库如此频繁增删改查的情况，很明显不适合，所以只能用链表作为索引的基础结构，剩下的就是解决链表的查询效率问题。

# 从链表到二叉树

使用链表作为存储结构，就必须要解决链表的查询效率问题，这时我们可以通过算法来优化效率，这里就不得不说二分查找法，利用二分法思维把链表拆成 N 对半分的节点，随后形成一个数据结构叫 二叉查询树，使用二分法思维衍生的树大大提升查询的性能。

# 从二叉查询树到平衡二叉树

即使二叉查找树相对于链表查询性能更高，但是还有缺陷，就是二叉树的高度不可控，如下图

树的节点变成线性结构，这样查询效率会很低，所以我们还需要优化算法来保证二叉树节点的平衡，使得节点高度差不会拉开；此时就衍生了平衡算法，最终变成 AVL 树和红黑树等树结构，也称为平衡二叉树，平衡二叉树通常会保证树的左右两边的节点层级相差不会大于 2。

# 从平衡二叉树到 B 树

平衡二叉树似乎接近理想状态了，但仍有优化空间，观察上面的树，不难发现这棵树的查询效率取决于树的高度，树越高，那么效率就会越低，那么我们只要减少树的层级，那么就可以提高树的查询效率。

所以 B 树相对于平衡二叉树最大的改变，就是 B 树的每个节点可存储的关键字增多了，特别是在 B 树应用到数据库的时候，节点存储关键字的数量充分利用磁盘块 IO 的原理 (磁盘数据存储是采用块的形式存储的，每个块大小为 4K，每次 IO 进行数据读取时，同一个磁盘块的数据可以一次性读取出来)，B 树只要把节点大小限制在磁盘块大小范围内，这样就可以只需一次 IO 就读取到节点所有数据；这样使得节点存储了更多的关键字，但是并不影响 IO 读取的次数，也因为单节点的存储关键字变多了，节点就会减少，那么树的层级也会减少，也就提高了检索的效率。

# 从 B 树到 B + 树

原以为 B 树已经是完全体了，却还有能优化的地方

  1. B 树的节点同时保存了索引的 key 和数据值（B 树的节点只保存索引 key 不保存值，使得索引树的层级进一步减少）
  2. 因为每个节点保存了数据值，这样查询不同的数据效率会显得不稳定，有的数据在树的顶上，而有的数据在树叶子
  3. 查询所有数据时，需要遍历整个 B 树的节点

而 B + 树在 B 树的基础上对以上问题进行优化：

  1. B + 树的非叶子节点不保存实际的数据，只保存索引 key，所有数据将保存到叶子节点
  2. B + 树叶子节点的关键字从小到大有序排列，左边结尾数据都会保存右边节点开始数据的指针

以上优化后相对于 B 树有以下优点：

  1. 树层级变少
  2. 查询更稳定
  3. 遍历整个树更快 ：因为所有数据全在叶子节点，所以只需要遍历所有叶子节点即可
  4. 排序和范围查询更方便 ：B + 树所有的叶子节点数据构成了一个有序链表，这样在进行数据排序和询范围大小查询数据的时候更方便，数据紧密性也更高

总结来说，从平衡二叉树、B 树、B + 树，总体来看它们的贯彻的思想是相同的，在链表的基础上，如何采用二分法和数据平衡策略来提升查找数据的速度。不同点是他们一个一个在演变的过程中通过 IO 从磁盘读取数据的原理进行一步步的演变，每一次演变都是为了让节点的空间更合理的运用起来，从而使树的层级减少达到快速查找数据的目的，在这个基础上，B + 树的查找速度快、性能稳定、排序快、扫表快等诸多特点也就让 Mysql 选择了 B + 树来作为索引的存储结构。

MySQL 的索引树一般都在 1-3 层结构，这不包括叶子节点层，经过计算表数据在千万级以内，都在 1-3 层
