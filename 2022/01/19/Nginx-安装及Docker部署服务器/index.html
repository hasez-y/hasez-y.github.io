# Nginx

Nginx 是一个高性能的 HTTP 和反向代理 web 服务器，特点是占用内存少，并发量高

# 正向代理

VPN 就是一个正向代理，代理客户端，客户端也就是我们的设备；

我们想在大陆访问外网时，就需要借助 VPN，我们请求 VPN 服务器，VPN 服务器再请求我们想要访问的外网网址，网站返回到 VPN 服务器，再返回到我们客户端上，实现了正向代理。

# 反向代理

代理服务端，有一台服务器负责接收请求并转发请求到其他服务器，实现负载均衡，也实现隐藏真实服务器的目的。

# 负载均衡

Nginx 提供的负载均衡策略有两种

  1. 内置策略 ：轮询，加权轮询，IP hash
  2. 拓展策略

如果想使用 Nginx 来解决 session 问题，可以使用 IP hash 策略，一般而言还是用 Redis 解决更合适

# 动静分离

一些静态资源类似 js、图片等资源，可以直接放到 Nginx 服务器中，不需要走到服务端，节省资源，也提高响应速度。

# 安装 Nginx

这里我们使用 Docker 来安装 Nginx，拉取最新镜像

# 运行 Nginx

参数说明

  • --name nginx-test 指定镜像运行后的容器名
  • -p 8080:80 端口映射，将本地 8080 映射到容器内部 80 端口
  • -d 设置容器后台运行

运行好后会返回该容器的 id，后续可以给容器打 tag 标识

# 测试

访问外网 ip+8080 即可访问到 nginx 服务器，前提是把防火墙 8080 端口打开。

# Nginx 服务部署

需要将 Nginx 容器内的配置文件映射到本地目录

先创建本地目录来存放 Nginx 相关文件

三个文件夹的作用

  • www 目录将映射 nginx 容器配置的虚拟目录
  • logs 目录将映射 nginx 容器的日志目录
  • conf 目录里的配置文件映射为 nginx 容器的配置文件

拷贝容器内 Nginx 默认配置文件到本地当前目录下的 conf 目录

先用 ps 查看容器 id，再通过 id 指定容器内目录复制到本地

开始部署

-v 用于挂载容器目录， : 前是本地目录， : 后是容器目录

# 验证

进入 /home/nginx/www 目录，也就是放静态资源的目录

随后访问 服务器公网 IP + 设置的端口 就可以访问到 Nginx 服务器静态资源了

除此之外 Nginx 还可以修改 conf 配置文件实现反向代理，后续有时间会另外发文章实现反向代理和负载均衡🤗
