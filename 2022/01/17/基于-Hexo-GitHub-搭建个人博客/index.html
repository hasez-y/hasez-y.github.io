# 基于 Hexo & GitHub 搭建个人博客

Hexo 是一款基于 Node.js 的静态博客框架

使用 Hexo 生成的静态文件，可以放在 GitHub 上托管，使用 GitHub Pages 来实现在公网访问博客。这套组合拳下来可以实现零代码、零成本搭建个人博客，快乐白嫖。

下面会分别介绍上述出现的名词

# Hexo

如官网所述 Hexo 是一个快速、简洁且高效的博客框架

使用 Hexo 前提是需要安装其他的应用程序

  • Node.js
  • Git

# Node.js

Hexo 是基于 nodeJS 编写的，所以需要安装一下 nodeJs 和里面的 npm 工具

官网下载地址: https://nodejs.org/zh-cn/

这里我们选择左边的长期维护版下载即可，下载后安装即可。

# Git

Git 是目前世界上最先进的分布式版本控制系统，我们这里需要使用 Git 来上传代码到 GitHub 代码仓库。

Git 官方下载地址: https://gitforwindows.org/ ，Download 下载后安装即可

# 安装 Hexo

Git 和 Node.js 安装好后就可以安装 Hexo 了，Hexo 会自动生成项目，所以我们需要先建好文件夹，进入一个文件夹后在路径打 cmd 回车 即可进入 Window 命令行，如图

输入 cmd 后回车会弹出命令行并自动 cd 到当前目录

    在命令行以下命令回车即可安装淘宝镜像

    读条读完即下载完成

  2. 使用 npm 安装 Hexo

  3. 安装 Hexo 后生成博客文件夹，下面命令的 myblog 可以自己改成想要的名字作为文件名

    然后会看到目录会多出一个文件夹

  4. cd 到目录，初始化文件夹 这里 myblog 就是文件夹名称，表示进入该目录

  5. 启动 Hexo

  6. 浏览器访问 http://localhost:4000/ 即可

    如果出现该页面，恭喜你，个人博客创建成功，下面会介绍如何上传博客到 GitHub、换主题、写博客、上传博客、Hexo 详细参数配置。

  7. 在命令行 Ctrl + C 组合键，然后输入 y 回车 就可以关闭 Hexo

当关闭 Hexo 时，再访问 http://localhost:4000 / 时是无法访问的

这时候就可能会觉得这个博客很鸡肋了啊，怎么还要我 24 小时运行啊，那我电脑不关机才能访问博客网站？

除了这个问题还有一个问题，那就是你在这台电脑启动 Hexo 并访问 http://localhost:4000 / 确实没问题，但是换台设备，用手机或让朋友访问这个地址，就发现他们无法访问，这也是一个问题；

GitHub 就是来解决上述两个问题的，将博客托管到 GitHub 上这样就能 24 小时运行还能公网访问。下面会详细介绍怎么使用 GitHub

# GitHub

GitHub 作为世界最大的同性交友网站（因为都是程序员），超 5 千万开发者正在使用。它是一个代码托管平台。

因为只支持 Git 作为唯一的版本库格式进行托管，故名 GitHub。

所以我们之前安装的 Git 这时候就有用了

# 注册 GitHub 账号

GitHub 官网 https://github.com/，GitHub 的服务器在海外，访问可能会有点慢，也可能访问不了，访问不了的话可能需要科学上网了。

进入官网右上角 Sign up 注册账号，用户名尽量用英文名

# 创建仓库

创建一个和你用户名相同的仓库，后面加.github.io，只有这样，将来要部署到 GitHub page 的时候，才会被识别，也就是 xxxx.github.io，其中 xxx 就是你注册 GitHub 的用户名，这里我仓库已经创过了所以才报错的。

# 生成 SSH 添加到 GitHub

之前安装了 Git，在文件夹右键会显示 Git Bash，打开即可

在命令行输入

随后在 C 盘 \ 用户 \ 当前用户 文件夹下找到 .ssh 文件夹

进入.ssh 目录会有 id_rsa 文件，这是你这台电脑的私人密钥，不能给其他人看。 id_rsa.pub 是公共密钥，可以随便给人看。把这个公钥放在 GitHub 上，这样当你链接 GitHub 自己的账户时，它就会根据公钥匹配你的私钥，当能够相互匹配时，才能够顺利的通过 git 上传你的文件到 GitHub 上

这里需要输入你的公钥，进入.ssh 用记事本方式打开 id_rsa.pub

记事本打开 Ctrl + a 全选，Ctrl + c 复制即可

Ttile 随便取，将 id_rsa.pub 的公钥粘贴进去，然后提交即可

提交后 会显示

# 将 hexo 部署到 GitHub

回到 Hexo 自动生成的文件夹 myblog 找到 _config.yml 配置文件

如果有代码编辑器可以用编辑器打开，没有的话用记事本打开即可，然后翻到最后面

将最后几行 deploy 修改成以下即可，需要改下用户名那两段

修改完成后记得 Ctrl + S 快捷键来保存再关闭记事本

这时候还需要安装 deploy-git 才能部署到 GitHub，回到 cmd 命令行输入命令读条读完即可

输入以下命令开始部署

  • hexo clean 清除了你之前生成的东西，也可以不加。
  • hexo generate 顾名思义，生成静态文章，可以用 hexo g 缩写
  • hexo deploy 部署文章，可以用 hexo d 缩写

部署时可能需要你输入 GitHub 用户名和密码

以上步骤完成后，你可以进入 GitHub 查看你的仓库，代码是否上传成功了

# GitHub Pages

用 GitHub Pages 显示仓库主页

点击 save 保存即可

随后访问 http:// 你的 github 用户名.github.io/ 即可

我的仓库地址为 http://hasez-y.github.io/

到这里已经成功将博客托管到 GitHub 实现公网访问，可以在任何设备浏览器输入该网址来访问你的博客

例外可以看看一些博客主题: https://hexo.io/themes/

这些主题都可以预览，也可以替换为自己的主题。

关于博客 更换主题、写博客、上传博客、配置博客配置参数，比如博客头像、博客名等等会另外再出一篇文章
