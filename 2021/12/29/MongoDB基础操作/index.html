MongoDB基础

为什么需要MongoDB

为了应对互联网三高而使用

  • 高并发
  • 海量数据
  • 高可用、高拓展

简介

MongoDB是一个开源、高性能、无模式的NoSql数据库，也是最像关系型数据库的非关系型数据库

支持的数据结构非常松散，是一种类似于JSON格式的BSON

下载地址: https://www.mongodb.com/try/download/community

启动

  • 启动前需要创建存放数据的文件夹

  • 切入bin目录启动并指定存放数据的文件夹

    # 注意路径是相对路径，文件夹是第一步创建的文件夹位置
mongod --dbpath=..\database\db

  • 客户端连接

    # 还是在bin目录下
mongo

除了命令行模式还有图形化界面客户端，Compass

常用命令

# 创建或选择数据库， 如果数据不存在则自动创建
use 数据库名称
# 查询所有数据库
show dbs
# 删除数据库, 选择需要删除的数据库
db.dropDatabase()

当使用use 创建数据库时是在内存中，在写入数据后才会写入磁盘，否则不会显示在dbs列表

集合创建

集合类似于mysql的表

# 集合创建分为显示和隐式
# 显示创建
db.createCollection("name")
# 查询所有集合
show collections
# 删除集合
db.name.drop()
# 隐式创建，直接创建文档时会自动创建集合

文档增删改查

类似于mysql的记录

# 单个插入文档
db.集合名.insert({"key": "value"})
# 查找
db.集合名.find()
# 多个插入文档
db.集合名.insertMany([{"key1": "value1"}, {"key2": "value2"}])

# 条件查询, 传入json为条件
db.集合名.find({"key": "value"})
# 查询单条数据库
db.集合名.findOne({key: "value"})
# 投影查询, 查询结构只返回部分字段
db.集合名.find({key: "value"}, {articleid: 1})

# 覆盖修改, 注意这个更新会直接全覆盖，如果只想更新文档的一个字段请用局部修改
db.集合名.update({条件}, {修改语句})
db.集合名.update({_id: "1"}, {name: "更新后的name"})
# 局部修改
db.集合名.update({_id: "1"}, {$set:{name: "更新后的name"}})
# 注意mongodb默认修改只修改符合条件的第一条数据, 如果需要全部修改
# 修改所有符合条件的数据, 添加{multi:true}
db.集合名.update({_id: "1"}, {$set:{name: "更新后的name"}}, {multi:true})

# 列值自增 $inc:
db.集合名.update({_id: "1"}, {$inc: {count: NumberInt(1)}})

# 删除文档
db.集合名.remove({条件})

分页查询

# 统计集合所有记录(文档)
db.集合名.count()
# 按条件统计记录数
db.集合名.count({条件})
# 查询前n条数据
db.集合名.find().limit(n)
# 跳过前y条记录
db.集合名.find().skip(y).limit(n)

# 排序查询 1为升序 -1为降序, 支持多个条件排序，按先后顺序
db.集合名.find().sort({需要排序的key: 1})

包含查询

# 包含查询 in
db.集合名.find({key: {$in: ["", ""]}})
# 不包含查询 nin
db.集合名.find({key: {$nin: ["", ""]}})

条件连接查询

# and、 or查询
db.集合名.find({$and: [{条件一}, {条件二}]})
db.集合名.find({$or: [{条件一}, {条件二}]})

索引的使用

mongodb索引数据库为B-Tree，Mysql则是B+Tree

使用索引会在海量数据中提高执行效率

# 查看索引
db.集合名.getIndexes()
# 创建索引 1为升序，-1为降序
db.集合名.createIndex(keys, options)
db.集合名.createIndex({user_id: 1})
# 复合索引
db.集合名.createIndex({user_id: 1, key2: -1})
# 移除索引, 可以指定索引名删也可以指定规则删
db.集合名.dropIndex({user_id: 1})

执行计划

# 用于分析查询效率或建立的索引是否生效
db.collection.find(query, options).explain(options)
# 是否使用索引看结构是COLLSCAN还是FETCH抓取
